# 头脑风暴：AI 驱动的活世界（AI Living World）

> 📅 2026-03-29 头脑风暴记录
>
> 本文档记录了关于 ADV.JS 未来发展方向的创意讨论，核心愿景是将传统视觉小说引擎升级为一个 **AI 驱动的、可交互的活世界**。

---

## 一、核心愿景

### 从「剧本驱动」到「角色驱动」

传统 ADV 游戏的对话和剧情由创作者预先编写。新的愿景是：

- **设定角色的初始性格和出身背景**（人物卡 `.character.md`）
- **由 AI 根据设定自动续写故事**，角色会根据性格和背景自主发展
- 设定一个场景（时间、地点、人物、大致事件），AI 即可生成该场景的完整内容
- 角色的**经历会反过来更新其性格/背景**（动态角色成长）

### 与传统动画/动漫的区别

| 维度       | 传统动画/动漫        | AI Living World                |
| ---------- | -------------------- | ------------------------------ |
| 故事走向   | 固定，由编剧决定     | 动态，由 AI + 玩家选择共同决定 |
| 角色发展   | 线性，预设的性格变化 | 非线性，基于经历动态演化       |
| 观众角色   | 被动观看             | 可以参与互动、引导故事         |
| 选项与分支 | 无                   | 提供选项，也可添加引导         |
| 可玩性     | 单次观看             | 无限重复，每次不同             |

## 二、关键创意

### 2.1 专业角色 NPC — 真正有知识的 AI 居民

不同于传统游戏中只有格式化回答的 NPC，AI Living World 中的角色可以拥有**真实专业知识**：

| 角色类型      | 能力                       | 与传统游戏 NPC 区别              |
| ------------- | -------------------------- | -------------------------------- |
| 🏥 医学专家   | 基于医学语料库回答健康问题 | 不是"你可以去医院看看"的模板回复 |
| ⚖️ 法律专家   | 基于法条、案例分析法律问题 | 能引用具体法条，分析具体案例     |
| 🏠 生活管家   | 日常生活百科知识           | 类似 AI 助手功能嵌入游戏角色中   |
| 💆 心理咨询师 | 情感倾听和心理疏导         | 角色化的情感陪伴，更有代入感     |

**实现思路**：

- 角色卡 `.character.md` 中定义角色的知识领域和人设
- 通过 RAG（检索增强生成）或专业知识库为角色注入真实知识
- AI 在回答时同时保持角色人设和专业性

### 2.2 文本小世界 — 自运转的生态

想象一个文本世界：

- 世界中有**剑士、普通人、卖菜大叔**等各式各样的角色
- 他们有自己的生活：会生病、会遇到纠纷、会有喜怒哀乐
- 角色之间会自然地产生互动和故事
- 世界按照自己的规则运转，**不需要玩家介入也会发展**

### 2.3 玩家的身份与介入方式

一个关键问题：**真人玩家如何进入这个小世界？**

可能的方案：

| 方案            | 描述                                             | 优缺点                       |
| --------------- | ------------------------------------------------ | ---------------------------- |
| 🎮 游戏角色模式 | 玩家捏一个角色，以角色身份进入世界参与互动       | 沉浸感强，但需要角色设计系统 |
| 👁️ 上帝视角模式 | 玩家作为观察者，可以在任意时刻介入引导故事走向   | 灵活度高，适合创作者         |
| 💬 对话访客模式 | 玩家可以随时与世界中的任意角色对话，但不属于世界 | 最简单，适合咨询类需求       |
| 🔄 混合模式     | 以上模式自由切换                                 | 最完整，但实现复杂度最高     |

### 2.4 人生模拟 — 平行人生

> "捏一个自己出来，看不同选择下的不同人生"

这是最有共鸣的创意之一：

- 玩家创建一个**以自己为原型的角色**
- 设定当前人生的关键节点（读什么专业？去哪个城市？和谁在一起？）
- AI 模拟不同选择下的人生轨迹
- 可以**同时运行多条平行人生线**，对比不同选择的结果
- 类似「人生重来模拟器」，但由 AI 驱动，更加丰富和真实

## 三、用户需求分析

用户玩游戏/看内容的核心动机：

| 需求类型    | 描述                 | 对应的世界功能                 |
| ----------- | -------------------- | ------------------------------ |
| 🎉 娱乐消遣 | 有趣、新奇、好玩     | 奇幻世界冒险、有趣角色互动     |
| 💗 情感陪伴 | 寻找情感慰藉         | 心理咨询师角色、陪聊角色       |
| 📚 知识获取 | 学到有用的东西       | 专家角色（法律、医学、生活等） |
| ⏰ 打发时间 | 轻松消磨时间         | 可以随时进入的轻互动           |
| 🔮 人生探索 | 探索不同选择的可能性 | 平行人生模拟                   |

## 四、产品形态思考

最终产品不限于某一种形态，可以是：

- **📖 可交互的书**：文本为主，随时可以和角色对话
- **🎮 游戏界面**：带立绘、背景、音乐的完整 AVG 体验
- **💬 聊天界面**：类似即时通讯，直接与世界中的角色对话
- **🌐 多形态融合**：根据场景自动切换展现形式

ADV.JS 已有的架构（主题系统 + 客户端渲染 + 多种插件）天然支持多形态。

## 五、技术方向与 TODO

### 5.1 角色系统增强

- [ ] **动态角色状态**：扩展 `.character.md` 支持动态属性（心情、健康、关系值等）
- [ ] **角色记忆系统**：角色记住与玩家的对话历史和关键事件
- [ ] **性格演化引擎**：基于经历自动更新角色的性格描述
- [ ] **知识注入机制**：为专业角色注入领域知识（RAG / 知识库）

### 5.2 世界模拟系统

- [ ] **世界时钟**：世界有自己的时间流逝，事件按时间线触发
- [ ] **角色自治 AI**：角色在没有玩家介入时也会自主行动
- [ ] **事件生成器**：基于世界设定随机/智能生成事件（纠纷、疾病、节日等）
- [ ] **角色关系网络**：定义和动态更新角色间的关系

### 5.3 玩家交互系统

- [ ] **角色创建器**：玩家捏脸/捏人生的系统
- [ ] **多视角切换**：在不同角色间切换视角观察世界
- [ ] **介入与引导**：玩家可以在关键时刻做出选择或提供引导
- [ ] **平行人生模式**：分支对比和平行运行

### 5.4 专业能力集成

- [ ] **知识库对接**：支持外部知识库接入（法律数据库、医学百科等）
- [ ] **角色能力验证**：确保专业角色的回答准确可靠
- [ ] **多模型支持**：不同角色可以使用不同的 AI 模型或 prompt 策略

### 5.5 与现有架构的融合点

| 现有模块               | 融合方式                                 |
| ---------------------- | ---------------------------------------- |
| `.character.md` 角色卡 | 扩展为动态角色卡，支持状态和记忆字段     |
| `world.md` 世界观      | 扩展为世界规则引擎的配置文件             |
| `@advjs/parser`        | 扩展解析器支持动态内容和条件分支         |
| `@advjs/core`          | 增加世界模拟循环和角色自治逻辑           |
| `plugin-openai`        | 作为 AI 生成的底层接口，支持多角色多模型 |
| MCP Server             | 暴露世界状态查询和操作接口               |
| Studio 移动端          | 作为随时随地与世界角色对话的入口         |

## 六、灵感参考

- **AI Town**（斯坦福 AI 小镇实验）：多 AI Agent 在虚拟小镇中自主生活
- **Westworld**（西部世界）：AI 角色拥有自己的故事线和记忆
- **The Sims**（模拟人生）：生活模拟 + 自主行为系统
- **Character.AI**：与 AI 角色对话，但缺少世界背景和角色间互动
- **NovelAI / AI Dungeon**：AI 驱动的文本冒险，但缺少视觉呈现

ADV.JS 的独特优势在于：**完整的视觉小说引擎 + Markdown 创作体系 + 多端支持**，是最适合实现这个愿景的基础设施。

## 七、阶段性规划建议

### 近期（可立即开始）

1. 设计**动态角色卡格式**（扩展 `.character.md`），支持状态、记忆、知识域字段
2. 实现**角色对话 API**：基于角色卡 + 对话历史生成角色回复
3. 在现有 `adv-story` skill 中实验**角色自主对话**

### 中期（1-3 个月）

4. 实现**世界时钟 + 事件系统**原型
5. 实现**玩家角色创建器**
6. 开发**平行人生模式**原型
7. 专业角色知识库 POC（选一个领域验证）

### 远期（3-6 个月）

8. 完整的**角色自治 AI 系统**
9. 多角色**自发互动**和关系演化
10. **多端体验**：桌面深度体验 + 移动端轻量对话
11. 开放**世界模板**，让创作者定义自己的 AI 小世界
